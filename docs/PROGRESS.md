## ✅ Done — Self-service password reset for all three auth planes (migration 0034)
- **Forgot-password → emailed token → set-new-password on business, PF, and client** (none had it before; only an admin-set-password on the client plane). One shared **definer-only** `password_reset_token` table (`plane` discriminator), modelled on `pf_link_token`: RLS forced, no policy/grant, touched only by SECURITY DEFINER fns (the flow is **pre-auth** — no session/RLS context at request or consume). Token is **sha256-hashed at rest** (raw only in the email link), **single-use** (`used_at`), **expiring** (`PASSWORD_RESET_TTL_MIN`=60); a new request invalidates prior live tokens.
- **Consume definers are atomic (owner-rights):** `pwreset_consume_business/_pf/_client(token_hash, new_hash)` spend the token, set `password_hash`, **revoke ALL the account's live refresh tokens** (a reset kills every session), and audit (`audit_log` / `pf_audit_log`); the plane discriminator blocks cross-plane consumption. `client_reset_lookup` resolves a client (login_id may be a student id) to the party contact email.
- **No enumeration incl. timing:** `request-reset` always returns generic `{ok:true}` and the lookup→mint→email runs **off the response path** (fire-and-forget) so an existing account isn't observably slower; `reset` failures collapse to one generic 400; a client with a non-email address is treated as not-found. No auto-login (redirect to the plane's login). Rate-limited (best-effort in-process): per-IP + per-identifier on request, per-IP on reset.
- **API:** `PasswordResetService` in `AuthModule` (exported); `@Public POST {/auth,/pf/auth,/client/auth}/request-reset` + `/reset` (PF/client keep `@PfRoute`/`@ClientRoute`). **Web:** "Forgot password?" link + `forgot-password` + `reset-password` pages per plane, six BFF routes, `middleware.ts` allowlists the new public pages.
- **Tests/reviews:** qa password-reset HTTP **17/17** (request→email-with-token; reset→old-pw-401/new-pw-200; single-use 400; forged/expired 400; pre-reset refresh dies; unknown→same generic 200 + no email; <8-char 400; @Public; client login_id→contact email) + regressions auth-http 15 / pf-core 7 / client-portal-http 18 unchanged; build + guard + migrate 0034 green.
